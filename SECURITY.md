Политика безопасности - Бот расписания школы
Обзор
В этом документе описаны меры безопасности, реализованные в Боте расписания школы для защиты данных пользователей и предотвращения несанкционированного доступа.

Аутентификация и авторизация
Регистрация пользователей
Проверка данных: Все данные регистрации проверяются:

ФИО должно содержать только буквы и состоять из 2-20 символов в каждой части

Названия классов должны соответствовать определенным шаблонам (5A-9B, 10П, 10Р, 11Р)

Максимум 30 пользователей на класс для предотвращения злоупотреблений

Доступ администратора
Проверка администратора: Привилегии администратора предоставляются на основе совпадения имени пользователя Telegram с предопределенным списком

Логирование безопасности: Все действия администратора логируются с ID пользователя и деталями

Управление сессиями: Состояния администратора сохраняются во время многошаговых операций

Защита данных
Санитизация ввода
Экранирование HTML: Весь пользовательский контент правильно экранируется для предотвращения XSS-атак

Предотвращение SQL-инъекций: Использование параметризованных запросов для всех операций с базой данных

Проверка типа файлов: Загрузка Excel файлов проверяется по расширению (.xlsx, .xls)

Ограничение запросов
Ограничение частоты: Максимум 20 запросов в минуту на пользователя

События безопасности: Нарушения лимита запросов логируются как события безопасности

Дедупликация обновлений: Обработанные обновления отслеживаются для предотвращения повторных атак
